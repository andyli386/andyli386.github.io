Sentiment analysis with TFLearn

In this notebook, we‚Äôll continue Andrew Trask‚Äôs work by building a network for sentiment analysis on the movie review data. Instead of a network written with Numpy, we‚Äôll be using TFLearn, a high-level library built on top of TensorFlow. TFLearn makes it simpler to build networks just by defining the layers. It takes care of most of the details for you.

We‚Äôll start off by importing all the modules we‚Äôll need, then load and prepare the data.

Preparing the data

Following along with Andrew, our goal here is to convert our reviews into word vectors. The word vectors will have elements representing words in the total vocabulary. If the second position represents the word ‚Äòthe‚Äô, for each review we‚Äôll count up the number of times ‚Äòthe‚Äô appears in the text and set the second position to that count. I‚Äôll show you examples as we build the input data from the reviews data. Check out Andrew‚Äôs notebook and video for more about this.

Counting word frequency

To start off we‚Äôll need to count how often each word appears in the data. We‚Äôll use this count to create a vocabulary we‚Äôll use to encode the review data. This resulting count is known as a bag of words. We‚Äôll use it to select our vocabulary and build the word vectors. You should have seen how to do this in Andrew‚Äôs lesson. Text to vector function

Now we can write a function that converts a some text to a word vector. The function will take a string of words as input and return a vector with the words counted up. Here‚Äôs the general algorithm to do this:

  • Initialize the word vector with np.zeros, it should be the length of the vocabulary.
  • Split the input string of text into a list of words with .split(' ').
  • For each word in that list, increment the element in the index associated with that word, which you get from word2idx.

Note: Since all words aren‚Äôt in the vocab dictionary, you‚Äôll get a key error if you run into one of those words. You can use the .get method of the word2idx dictionary to specify a default returned value when you make a key error. For example, word2idx.get(word, None) returns None if word doesn‚Äôt exist in the dictionary.

Building the network

TFLearn lets you build the network by defining the layers.

Input layer

For the input layer, you just need to tell it how many units you have. For example,

would create a network with 100 input units. The first element in the list, None in this case, sets the batch size. Setting it to None here leaves it at the default batch size.

The number of inputs to your network needs to match the size of your data. For this example, we‚Äôre using 10000 element long vectors to encode our input data, so we need 10000 input units.

Adding layers

To add new hidden layers, you use

This adds a fully connected layer where every unit in the previous layer is connected to every unit in this layer. The first argument net is the network you created in the tflearn.input_data call. It‚Äôs telling the network to use the output of the previous layer as the input to this layer. You can set the number of units in the layer with n_hidden, and set the activation function with the activation keyword. You can keep adding layers to your network by repeated calling net = tflearn.fully_connected(net, n_units).

Output layer

The last layer you add is used as the output layer. There for, you need to set the number of units to match the target data. In this case we are predicting two classes, positive or negative sentiment. You also need to set the activation function so it‚Äôs appropriate for your model. Again, we‚Äôre trying to predict if some input data belongs to one of two classes, so we should use softmax.
